# Огляд Вимог

## Мета та Область Застосування
Wellity — це кросплатформенний мобільний застосунок для фітнесу та відстеження звичок, який допомагає користувачам контролювати свої звички, виконувати тренування, формувати здорові щоденні ритуали та візуалізувати прогрес через статистику та аналітику. Застосунок забезпечує соціальну взаємодію та обмін досягненнями між користувачами, підтримуючи мотивацію та заохочуючи регулярне виконання завдань.

## Глосарій
- **Habit Tracker** — структурований інструмент для моніторингу параметрів звичок з шаблонами полів (числові, текстові, оцінки)
- **Нотатки (Notes)** — вільні текстові записи користувача без структури
- **Workout Session** — активне виконання тренування користувачем з таймером та подальшою оцінкою
- **Соціальні функції** — можливості взаємодії користувачів, включаючи обмін досягненнями та перегляд профілів
- **Модуль тренувань** — функціонал для фітнесу, включаючи відстеження вправ та статистику
- **Адаптивні тренування** — система автоматичної зміни навантаження на основі оцінок користувача
- **Верифіковане тренування** — тренування, що пройшло перевірку якості та отримало спеціальний бейдж

## Функціональні Вимоги (FR)

### Автентифікація та профіль
- **FR-001:** Система реєстрації та автентифікації користувачів з підтримкою двох типів акаунтів (користувач, тренер)

### Habit Tracking
- **FR-002:** Створення, редагування та видалення особистих habit trackers користувачами з підтримкою шаблонів полів
- **FR-003:** Щоденне відстеження та фіксація виконання habit trackers
- **FR-015:** Система шаблонів для habit tracking (текстові поля, числові, оцінки) для структурованого відстеження параметрів

### Нотатки
- **FR-016:** Створення, редагування та видалення текстових нотаток у вільній формі
- **FR-017:** Прикріплення нотаток до тренувань та habit trackers для контекстуальної інформації

### Візуалізація та статистика
- **(НЕ АКТУАЛЬНО) FR-004:** Інтерактивна візуалізація прогресу в календарі
- **FR-005:** Панель статистики та аналітики тренувань з можливістю фільтрації за періодами
- **FR-007:** Відстеження та звітність рівня енергії користувача

### Тренування
- **FR-011:** Система верифікації тренувань з можливістю надання спеціального бейджа "Верифіковане тренування"
- **FR-012:** Управління доступом до тренувань (публічні безкоштовні, платні, по запрошенню)
- **FR-013:** Блокування паралельних workout sessions (один користувач може мати лише одну активну сесію одночасно)
- **FR-014:** Адаптивна система тренувань з динамічною зміною навантаження (підходи, вага, повторення) на основі оцінок складності від користувача

### Рекомендації та досягнення
- **FR-006:** Система персоналізованих рекомендацій з відновлення на основі інтенсивності тренувань
- **FR-008:** Система віртуальних нагород та досягнень (особисті badges за власні досягнення, глобальна стрічка досягнень інших користувачів)

### Профіль та соціальні функції
- **FR-009:** Управління профілем користувача (аватар, опис, статистика) та соціальні функції (перегляд публічних профілів інших користувачів, обмін досягненнями)
- **FR-010:** Push-сповіщення для нагадувань про habit trackers, тренування та досягнення

## Нефункціональні Вимоги (NFR)

### Ефективність Продуктивності (PERF)
- **PERF-001:** Час відповіді для фіксації habit trackers або завершення workout session має становити ≤ 1 секунди за нормальних умов навантаження
- **PERF-002:** Час запуску застосунку не повинен перевищувати 3 секунди на пристроях з мінімальними системними вимогами
- **PERF-003:** Візуалізація статистики має відображатися протягом 2 секунд для наборів даних до 365 днів
- **PERF-004:** Адаптивна система має генерувати налаштоване тренування протягом ≤ 3 секунд після отримання оцінки від користувача

### Надійність (REL)
- **REL-001:** Час роботи системи має становити ≥ 99.0% під час пікових годин використання (6:00-22:00)
- **REL-002:** Успішність синхронізації даних має становити ≥ 98% на всіх підтримуваних платформах
- **REL-003:** Застосунок має коректно обробляти переривання мережі без втрати даних; workout sessions мають зупинятися з попередженням користувача при втраті з'єднання
- **REL-004:** Workout session має синхронізуватися між пристроями користувача у реальному часі з затримкою ≤ 2 секунди

### Безпека (SEC)
- **SEC-001:** Паролі користувачів мають шифруватися з використанням промислових стандартів (мінімум SHA-256)
- **SEC-002:** Особисті дані користувачів мають передаватися лише через HTTPS з'єднання
- **SEC-003:** Тайм-аут сеансу автентифікації не повинен перевищувати 24 години неактивності
- **SEC-004:** Платні тренування мають обробляти платежі через захищені payment gateway з дотриманням PCI DSS compliance

### Здатність до Взаємодії (USAB)
- **USAB-001:** Нові користувачі мають мати можливість створити свій перший habit tracker або розпочати перше тренування протягом 5 хвилин після першого запуску застосунку
- **USAB-002:** Інтерфейс застосунку має підтримувати стандарти доступності (мінімум WCAG 2.1 AA)
- **USAB-003:** Критичні дії користувача мають виконуватися максимум за 3 дотики/кліки

### Сумісність (COMP)
- **COMP-001:** Застосунок має функціонувати на iOS 14.0+ та Android API рівень 24+ (Android 7.0)
- **COMP-002:** Функціонал експорту даних має підтримувати стандартні формати (CSV, JSON)
- **COMP-003:** Застосунок має підтримувати послідовну функціональність на всіх підтримуваних платформах
