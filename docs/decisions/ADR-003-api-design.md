## ADR-003: Дизайн API

## Статус
Прийнято (2025-10-01)

## Контекст</br>
- Функціональні вимоги (FR):</br>

  - FR-001: Автентифікація користувачів

  - FR-005: Панель статистики та аналітики тренувань

  - FR-006: Персоналізовані рекомендації з відновлення

  - FR-008: Система нагород та досягнень

  - FR-009: Управління профілем та соціальні функції

  - FR-010: Push-сповіщення

- Нефункціональні вимоги (NFR):</br>

  - PERF-001: Час відповіді ≤ 1 сек для локальних операцій

  - REL-002: Синхронізація даних ≥ 98%

  - SEC-001: Шифрування паролів (SHA-256)

  - SEC-002: Тільки HTTPS для передачі даних

  - SEC-004: PCI DSS для платежів

  - USAB-003: Критичні дії максимум за 3 кліки

## Рішення</br>
Використовувати REST API для Custom Backend, а Firebase SDK — напряму у </br> Flutter для auth, CRUD, realtime, notifications.

## Наслідки </br>

✅ Простота інтеграції з Flutter </br>
✅ Backend stateless → легке масштабування </br>
✅ Відповідність PCI DSS для платежів </br>
⚠ Дві моделі доступу (Firebase SDK + REST API) → потрібні guidelines </br>
❌ REST менш гнучкий за GraphQL </br>

## Розглянуті альтернативи:</br>

- GraphQL (відхилено)
❌ Надмірна складність.</br>

- gRPC (відхилено)
❌ Непотрібна складність для MVP. </br>
