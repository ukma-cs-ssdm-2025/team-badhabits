## ADR-002: Вибір бази даних

## Статус
Прийнято (2025-10-01)

## Контекст
- Функціональні вимоги (FR):

  - FR-002: CRUD для звичок

  - FR-003: Щоденне відстеження звичок

  - FR-004: Візуалізація прогресу в календарі

  - FR-005: Аналітика тренувань

  - FR-007: Відстеження рівня енергії

- Нефункціональні вимоги (NFR):

  - REL-002: Синхронізація даних ≥ 98%

  - PERF-001: Час відповіді для фіксації звичок ≤ 1 сек

  - COMP-001: Підтримка iOS 14+ та Android 7.0+

## Рішення
Використовувати Firebase Firestore для CRUD і offline-first та Firebase Realtime Database </br> для workout sessions і швидкої синхронізації.

## Наслідки
✅ Offline-first із коробки </br>
✅ Відповідність вимогам MVP (99% uptime, sync ≤2s) </br>
✅ Масштабування без адміністрування </br>
⚠ Vendor lock-in Firebase </br>
⚠ Складні запити (агрегації, аналітика)

## Розглянуті альтернативи

- PostgreSQL/MySQL (відхилено)</br>
❌ Потребує адміністрування та бекенду.</br>

- MongoDB (відхилено)</br> 
❌ Немає offline-first і realtime із коробки.</br>
